Fitness Components

This is a demo of the components with default and alternative styles.

The base design is based upon Apple Fitness rings. You can use a fully featured card or just the rings.

Cards

To use the card, you must create a fitness-card element with the relevant attributes representing the data.

The date is optional, if there is not set, then the card header is not rendered. steps, distance and flights are also optional â€“ if none are provided, then the card footer is not rendered.

The distance attribute should be provided in in meters. By default, this is converted to miles but you can change this to kilometres using units="km" attribute.

<fitness-card
    date="2023-06-04"
    steps="5258"
    distance="2455"
    units="miles"
    flights="3"
    move-total="1800"
    move-goal="1400"
    exercise-total="20"
    exercise-goal="30"
    stand-total="9"
    stand-goal="12"
></fitness-card>

Rings

To use the ring, you must create a fitness-ring element with the relevant attributes representing the data.

<fitness-ring
    move-total="1800"
    move-goal="1400"
    exercise-total="20"
    exercise-goal="30"
    stand-total="9"
    stand-goal="12"
></fitness-ring>
